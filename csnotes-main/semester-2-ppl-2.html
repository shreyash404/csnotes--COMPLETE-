Programming with Python-II


Python is a popular programming language that is widely used in the field of computer science. It is a high-level language that is easy to learn and use, and it has a wide range of applications, including web development, data analysis, scientific computing, and artificial intelligence.


The Programming with Python-II subject is an advanced course that builds on the concepts and skills learned in the Programming with Python-I course. It covers more advanced topics in Python programming, such as object-oriented programming, data structures, and algorithms.


In this course, students will learn how to design and implement complex Python programs that are efficient and scalable. They will also learn how to use various libraries and frameworks, such as NumPy and Pandas, to perform data analysis and scientific computing tasks.


To succeed in this course, students should have a strong foundation in computer science concepts and be comfortable with the Python programming language. They should also have a strong problem-solving skills and be able to think critically and logically.


Overall, the Programming with Python-II subject is an important course for computer science students to take as it helps them develop advanced programming skills and prepares them for a wide range of careers in the field.

In Python, a class is a template for creating objects. It defines the characteristics of the objects that will be created from it, and the methods that those objects can use. An object is an instance of a class, and it has the characteristics and methods defined by the class.

Inheritance is a way of creating a new class that is a modified version of an existing class. The new class is called the subclass, and the existing class is the superclass. The subclass inherits characteristics and methods from the superclass, and can also have additional characteristics and methods of its own.

There are several types of inheritance in Python, including:

  • 1. Single inheritance: This is when a subclass inherits from a single superclass.

  • 2. Multiple inheritance: This is when a subclass inherits from multiple superclasses.

  • 3. Multi-level inheritance: This is when a subclass inherits from a superclass, and that superclass itself inherits from another superclass. This creates a chain of inheritance.

  • 4. Hierarchical inheritance: This is when multiple subclasses inherit from a single superclass.

  • 5. Hybrid inheritance: This is a combination of multiple inheritance and multi-level inheritance.

Encapsulation is a technique for combining the code acting on the data (methods) and the data itself (variables) into a single entity. Encapsulation means that a class's variables are kept private from other classes and are only accessible through its own methods.

Python's Polymorphism introduces methods with the same name as those in the parent class in the child class. The parent class's methods are passed down to the child class through inheritance. Additionally, a method that a child class has inherited from a parent class may be changed.

In Object-Oriented Programming (OOP), built-in functions are functions that are pre-defined in the programming language and are available for use without the need for the programmer to define them. These functions are often related to the basic functionality of the programming language, such as input/output, data type conversion, and mathematical operations.

In Python, there are several built-in functions that are commonly used in object-oriented programming (OOP). Some of these include:

__init__(): This is a special method that is called when an object is created. It is used to initialize the object's attributes and perform any other setup that is required.
__str__(): This method is used to create a human-readable string representation of the object. It is called when the built-in print() function is used on the object, or when the str() function is used to convert the object to a string.
__eq__() : This method is used to define the behavior of the == operator when used with objects of the class.
__add__() : This method is used to define the behavior of the + operator when used with objects of the class.
__len__() : This method is used to define the behavior of the len() function when used with objects of the class.
__del__(): This method is called when an object is deleted. It is used to perform any necessary cleanup, such as freeing up resources that were allocated by the object.

These are just a few examples of the built-in functions that are available for use in Python OOP. There are many others, each with its own specific purpose and usage.

Single Inheritance:
Single inheritance is a mechanism in OOP where a derived class inherits properties and behaviors from a single, base class. The syntax for single inheritance in Python is as follows:

Syntax:
class Child(Parent):

Example:
class Dog(Animal):

Multiple Inheritance:
Multiple inheritance is a mechanism in OOP where a derived class inherits properties and behaviors from multiple base classes. The syntax for multiple inheritance in Python is as follows:

Syntax:
class Child(Parent1, Parent2, Parent3):

Example:
class Fish(Aquatic, Oviparous, Vertebrates):

Multi-level Inheritance:
Multi-level inheritance is a mechanism in OOP where a derived class inherits properties and behaviors from a base class, which in turn inherits from another base class. The syntax for multi-level inheritance in Python is as follows:

Syntax:
class Child(Parent):
class GrandChild(Child):

Example:
class Tiger(BigCats):
class BengalTiger(Tiger):

Hierarchical Inheritance:
Hierarchical Inheritance is a mechanism in OOP where multiple derived classes inherit from a single base class. The syntax for hierarchical inheritance in Python is as follows:

Syntax:
class Parent:
class Child1(Parent):
class Child2(Parent):

Example:
class Birds:
class Pigeons(Birds):
class Sparrows(Birds):

Hybrid Inheritance:
Hybrid Inheritance is a combination of more than one type of inheritance. It is a combination of one or more types of inheritance. The most common types of inheritance that are combined in Hybrid Inheritance are Multiple Inheritance and Multi-level Inheritance.

Syntax:
class Parent:
class Child1(Parent):
class Child2(Parent1, Parent2):

Example:
class Mammals:
class Dogs(Mammals):
class Bats(Mammals, WingedAnimals):

Threads are lightweight, independent units of execution that can run concurrently within a process. They have their own program counter, stack, and local variables. In Python, the threading module provides an easy way to create and manage threads.

Multi-threading is the ability of a central processing unit (CPU) (or a single core in a multi-core processor) to provide multiple threads of execution concurrently, supported by the operating system.

Python's threading module allows the creation of multiple threads within a program. Each thread runs parallel to the main thread, and can run a specific function or task. This allows for concurrent execution of multiple tasks, improving the overall performance of the program.

Here is an example of how to create and start a new thread in Python using the threading module:

import threading

def my_function():
# code to be executed in the new thread
new_thread = threading.Thread(target=my_function)
new_thread.start()

It is important to note that while multi-threading can improve performance, it can also lead to race conditions and other synchronization issues if not used carefully. Python's threading module provides a Lock class that can be used to synchronize access to shared resources.

In Python, the threading module provides a way to create and manage threads.

threading.activeCount() returns the number of thread objects in the caller's thread control.
threading.currentThread() returns a list of all thread objects in the caller's process.
threading.enumerate() returns a list of all thread objects that are currently active.
threading.Lock() creates a new lock object.
threading.RLock() creates a new reentrant lock object.
threading.Semaphore(value) creates a new semaphore object and initializes the internal counter to value.
threading.BoundedSemaphore(value) creates a new bounded semaphore object and initializes the internal counter to value.
threading.Event() creates a new event object.
threading.Barrier(parties, action=None, timeout=None) creates a new barrier object for parties number of threads.
threading.Timer(interval, function, args=[], kwargs={}) creates a new timer object that will run the function with specified arguments after interval seconds have passed.

You can use these functions to create and manage threads, synchronize access to shared resources, and set timers.

The Thread class in the python threading module has several methods that can be used to create and control threads. Some of the most commonly used methods are:

start(): This method is used to start a new thread. It calls the run() method on the thread object.
run(): This method is the entry point for a new thread. It should be overridden by the user to provide the code that the thread should execute.
join([time]): This method causes the calling thread to wait for the thread on which it is called to terminate. If a time argument is provided, the calling thread will wait for the specified number of seconds.
is_alive(): This method returns True if the thread is executing and False otherwise.
getName(): This method returns the name of the thread.
setName(): This method sets the name of the thread.
ident(): This attribute will return the thread identifier of the thread.
daemon(): This attribute tells whether the thread is daemonic thread or not.
isDaemon(): This method tells whether the thread is daemonic thread or not.
setDaemon(): This method changes the daemonic thread status of the thread.
enumerate(): This method returns a list of all thread objects in the caller's thread control.
activeCount(): This method returns the number of thread objects in the caller's thread control.
currentThread(): This method returns the current thread object, corresponding to the caller's thread of control.


The syntax for creating a thread in Python using the threading module is as follows:

import threading

def function_to_run_in_thread():
# code to run in thread

# Create the thread object
thread = threading.Thread(target=function_to_run_in_thread)

# Start the thread
thread.start()

Synchronizing threads in Python is the process of coordinating the execution of multiple threads such that they run in a specific order. This is done to ensure that multiple threads are working in a coordinated manner, and to prevent them from interfering with each other's execution. To synchronize threads in Python, we can use synchronization primitives such as locks, semaphores, and conditions.

The use of these primitives helps to ensure that the shared resources between threads are accessed in a controlled and coordinated manner, thus preventing race conditions and other issues that may arise from concurrent access. The threading module in Python provides several built-in synchronization primitives, which can be used to synchronize the execution of multiple threads.

A priority queue is a data structure that stores elements in such a way that the element with the highest priority (or lowest priority, depending on the implementation) is always accessible in constant time. This makes it useful in situations where you want to process elements in order of priority, such as scheduling tasks in a multi-threaded environment.

In Python, the queue module provides a PriorityQueue class that implements a priority queue. The important functions of the PriorityQueue class include:

PriorityQueue class in the queue module provides a priority queue.
Important functions of the PriorityQueue class:
put(item, priority): adds an item to the queue with the specified priority.
get(): retrieves and removes the item with the highest priority from the queue.
qsize(): returns the number of items in the queue.
empty(): returns a boolean indicating whether the queue is empty or not.

The PriorityQueue class is thread-safe and can be safely accessed by multiple threads at the same time.

However, locks or semaphores should be used to ensure the queue is updated atomically in a multi-threaded environment.

The read, readline, and readlines are methods in Python used to read data from a file. Here's a more detailed explanation of each:

1.read(size) - reads at most size bytes from the file and returns them as a string. If size is not specified, it will read all the remaining data in the file and return it as a string.
Example: 

 
with open("sample.txt", "r") as file:
    contents = file.read(5)
    print(contents)

2.readline() - reads the next line of the file and returns it as a string. If the file is empty, it returns an empty string.
Example: 
 
with open("sample.txt", "r") as file:
    line = file.readline()
    while line:
        print(line)
        line = file.readline()

3.readlines() - reads all the lines of the file and returns them as a list of strings.
Example: 
with open("sample.txt", "r") as file:
      lines = file.readlines()
      print(lines)

write(string) - The write() function is used to write a specified string to an open file. It writes the string to the file and returns the number of characters written. 

 
 def write_example():

def writelines_example():

1. r - The r mode is the default mode for file objects in Python. In this mode, the pointer is always positioned at the beginning of the file and can only be used for reading text. This is a read-only mode.

example:

2. r+ - The "r+" mode allows for both reading and writing in a text file. The pointer is positioned at the start of the file in this mode.

example:

3. rb - The "rb" mode is the default mode for binary files in Python. The "b" in "rb" indicates binary format and the "r" indicates that the file can only be read. The pointer is positioned at the beginning of the file in this mode.

example:

obj=open('name.txt','rb')

4. rb+ - The "rb+" mode allows for both reading and writing in binary format. The pointer is positioned at the start of the file in this mode.

example:

obj=open('name.txt','rb+')

5. w - The "w" mode opens a file for write-only access. If the file already exists, its contents will be overwritten. If the file does not exist, a new file will be created for writing.

example:

obj=open('name.txt','w')

6. w+ - The "w+" mode opens a file for both writing and reading. If the file already exists, its contents will be overwritten. If the file does not exist, a new file will be created for both writing and reading.

example:

obj=open('name.txt','w+')

7. wb - The "wb" mode opens a file in binary format for write-only access. If the file already exists, its contents will be overwritten. If the file does not exist, a new file will be created for writing in binary format.

example:

obj=open('name.txt','wb')

8. wb+ - When a file is opened in write and read mode with binary format, if it already exists, the existing content will be replaced. However, if the file does not exist, a new one will be created for writing.

example:

obj=open('name.txt','wb+')

9. a - When a text file is opened in append mode, if it already exists, new content will be added to the end without erasing the previous data. If the file does not exist, a new one will be created for writing. In both cases, the file pointer is positioned at the end of the file.

example:

example:

11. ab - When a binary file is opened in append mode, if it already exists, new content will be added to the end without altering the previous data. If the file does not exist, a new one will be created for writing. In both cases, the file pointer is located at the end of the file.



example:

If the file does not exist, a new one will be created for both reading and writing. The file pointer will be positioned at the end of the file if it already exists.

example:

An iterable can be traversed using the iter() and next() methods. The iter() function returns the object being iterated over each time the iteration takes place. The next() method moves the iteration to the next position and returns the value at that location. If the end of the iterable is reached, the next() method will raise a StopIteration exception.

example:
>>> collection=['we','Love','csnotes'] #defined an iterable of list type
>>> x=iter(collection) # x is an object of iter(), any variable name can be used in place of x
>>> print(next(x)) #use of next()
we
>>> print(x.__next__()) #calling next() with x obj
Love

In Python, exception handling involves managing interruptions in the normal flow of a program caused by exceptions. Unlike in Java, where the catch clause is used, Python uses the except clause to handle exceptions.

1. ArithmeticError : The base class for errors arising from numerical computations is referred to as this.

2. AssertionError : This exception is raised when assert statement fails

3. AttributeError : An error is triggered when the assignment or accessing of an attribute is unsuccessful.

4. EnvironmentError : Errors occurring outside of the Python environment are raised with this.

5. EOFError : An exception is thrown when there is no input obtained from input() or raw_input().

6. Exception : The base class for all exceptions is this, with all other exceptions being derived from it. In order to create custom exceptions, it must be imported, even for user-defined exceptions.

7. FloatingPointError : An error is triggered when a floating point calculation results in an error.

8. ImportError : An exception is raised when the import statement fails to execute.

9 . IndexError : An error is triggered when an index is not located within a sequence.

10. IOErrors : This exception is thrown in the following cases:
1.When an Input/Output operation fails to execute.
2.In the event of errors related to the operating system.


11. KeyboardInterrupt : An error is triggered when the program's flow is interrupted by the user pressing Ctrl+c.

12. KeyError : An error is triggered when a key is not located in a dictionary, resulting in a KeyError exception.

13. LookupError : This serves as the base class for all errors related to search or retrieval.

14. NameError : An error is triggered when an identifier is not located within the local or global namespace, resulting in an exception.

15. NotImplementedError : An error is raised when an abstract method, which is expected to be implemented in a derived class, is not properly implemented.

16. OverflowError : An exception is thrown when a calculation surpasses the maximum limit for a numeric data type.

17. RuntimeError : An error is raised in situations where it does not belong to any other specific category, resulting in this exception.

18. StandardError : This serves as the base class for all iterations, excluding StopIteration and SystemExit.

19. StopIteration : An error is triggered when the next() method of an iterator has reached the end of an iterable and has no further objects to reference, resulting in a StopIteration exception.

20. SyntaxError : An error is raised when there is an issue with the syntax in Python, resulting in a SyntaxError exception.

21. SystemError : An internal problem is detected by the Python interpreter, resulting in a SystemError exception being raised.

22. SystemExit : The sys.exit() function raises this exception, which, if not handled within the code, causes the interpreter to exit.

23. TypeError : An error is raised when an attempt is made to use a built-in function that is not applicable for the given data type, resulting in a TypeError exception.

24. UnboundLocalError : An error is triggered when an attempt is made to access a local variable within a function, but it has not been assigned a value, resulting in a UnboundLocalError exception.

25.ValueError : An error is raised when a function has a valid number of arguments, but they are of invalid values, resulting in an exception.

26. ZeroDivisionError : An exception is thrown when a division or modulo operation by zero is performed for any numeric data type, resulting in a ZeroDivisionError.
