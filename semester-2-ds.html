Data Structure


Data structures are ways of organizing and storing data in a computer so that it can be accessed and modified efficiently. Different types of data structures are suited to different kinds of applications, and some are highly specialized to specific tasks. Some common types of data structures include:


  • Arrays: An array is a collection of items that are stored in a contiguous block of memory. Arrays are indexed by a numerical key, and items can be accessed and modified by their position in the array.

  • Linked lists: A linked list is a collection of items that are stored in separate nodes and are linked together with pointers. Linked lists are useful for inserting and deleting items, but they are not as efficient as arrays for accessing items by their index.

  • Trees: A tree is a data structure that consists of a set of nodes connected by edges. Trees have a hierarchical structure, with a root node at the top and child nodes below it. Trees are often used to store data in a sorted order, and they are useful for searching, inserting, and deleting data.

  • Hash tables: A hash table is a data structure that uses a hash function to map keys to indices in an array. Hash tables are efficient for searching, inserting, and deleting data, and they are often used to implement associative arrays and databases.

    • Overall, data structures are an important concept in computer science because they provide the foundation for storing and organizing data in a way that is efficient and easy to use.

    Data abstraction is the process of hiding certain details and showing only essential information to the user. Data abstraction helps to reduce complexity and make the program easier to understand and maintain. It also helps to protect the data from unauthorized access or modification.

    There are several types of data abstraction, including:

    1. Procedural abstraction: A function is used to hide the implementation details of a specific task or algorithm.

    2. Data abstraction: A data type is used to represent a group of related data and hide the implementation details.

    3. Object-oriented abstraction: In object-oriented programming, a class is used to represent a group of related data and functions and hide the implementation details.

    4. Control abstraction: A control construct is used to hide the details of control flow in a program.

    5. Hardware abstraction: A hardware interface is used to hide the details of a specific hardware device and provide a uniform interface to the user.

    6. Abstract data type: An abstract data type (ADT) is a logical description of the operations that can be performed on a data type, without specifying the implementation details.

    7. Layer of abstraction: A layer of abstraction is a separation of functionality in a system, where each layer hides the details of the underlying layers.

    Preconditions specify the conditions that must be satisfied before an operation on a data structure can be performed. For example, a precondition for inserting an element into a sorted array might be that the array must be sorted. If the precondition is not met, the operation may not be allowed or may not produce the expected results.

    Postconditions specify the conditions that must be satisfied after an operation on a data structure has been performed. For example, a postcondition for inserting an element into a sorted array might be that the array remains sorted after the insertion. If the postcondition is not met, it may indicate that something went wrong during the operation.

    Preconditions and postconditions can be used to help ensure the correctness and reliability of a program by specifying the conditions under which an operation is allowed and the conditions that must be satisfied after the operation has been completed.

    There are two types of Data structure:
    1. Liniear Data Structure
    This particular data structure has a linear pattern in the way the data are organised. The linear arrangement of the data items creates a direct connection between each element and the ones before it and after it. The structure facilitates single-level data storage because the elements are stored linearly. Example : Stack , Queue , Array, Linked List
    Liniear Data Structure have more 2 types as follows:

    i. Static Data Structure
    The size of a static data structure is predetermined. Although the data structure's allotted memory space cannot be changed, its content can. Array is a good example of a static data structure.

    ii. Dynamic Data Structure
    In a dynamic data structure, the structure's size is not fixed and can change depending on the operations being carried out on it. Dynamic data structures are made to make it easier to update data structures while the programme is running.

    2. Non-Liniear Data Structure
    The data elements in this type of data structure change from a linear or sequential arrangement. There is not a single level involved because the data structure is non-linear. As a result, a user cannot visit every element of it in a single run
    Example : Trees and Graphs

    The map is an ADT that contains a collection of records. It states what all operations can be performed, but not their implementation.

    Every record of a map contains a key and a value.

    A value is mapped to an unique key.

    Map or dictionary is used to search a data using unique key value.

    The map is a container, which stores the record with unique key value.

    List and Array are the option to implement Map ADT. List is the best choice as it can expand automatically when required.

    We need to store the key component and corresponding value component for each entry in Map.

    The implementation is done by using following approach:
    ** Two list ** Single list
    In Two list approach, two lists are maintained. One is to store the keys and another to store the related values.

    In single list approach, key and corresponding data can be stored in a single object.
