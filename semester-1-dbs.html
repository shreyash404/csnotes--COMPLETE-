Database Management Systems


A database management system (DBMS) is a software that enables the creation, organization, and management of databases. It is an essential tool for storing, organizing, and manipulating large amounts of data in a structured and efficient way.


MySQL is a popular open-source DBMS that is widely used in computer science. It is a fast, reliable, and scalable DBMS that supports a wide range of data types and can handle large amounts of data.


One of the main benefits of using MySQL is that it allows users to easily create and manipulate databases using Structured Query Language (SQL). SQL is a programming language specifically designed for interacting with databases, and it is widely used in computer science for tasks such as querying, updating, and deleting data from a database.


In addition to its powerful SQL capabilities, MySQL also offers a range of features that make it easy to use and manage. It has a user-friendly interface, support for multiple users and security features, and a range of tools and utilities for backing up, restoring, and optimizing databases.


Overall, MySQL is an essential tool for computer scientists and professionals working with large amounts of data. Its powerful features and ease of use make it an excellent choice for a variety of applications, from web development to data analysis and more.

1. Data can be defined as facts that can be recorded and that have implicit meaning. For example, the names, telephone numbers, and addresses of people.
2. A Database is a collection of related data. It is a logically related collection of data with some inherent meaning. A database is designed for a specific purpose. It has an intended group of users and applications.
3. A DataBase Management System (DBMS) is a collection of programs that enables users to create and maintain a database.
Examples of DBMS - MySQL, Microsoft Access, Oracle, MongoDB etc. 4. The database and DBMS software together is called a Database system.

Advantages of DBMS -

Controlling Redundancy :- In traditional file processing, every user group maintains its own files. This redundancy in storing the same data multiple times leads to several problems. In DBMS, all data is integrated into a single database file.
Data Security :- DBMS can enforce access controls to different classes of users. This provides security for protecting database against accidental or intentional loss, destruction or misuse.
Data sharing :- DBMS can allow multiple users to access the database at the same time. Remote users also can share the same data. Same database can be shared between different applications.
Data Integrity :- Integrity stands for consistency and correctness. DBMS can enforce integrity constraints which provide a way of ensuring that changes made to the database by authorized users do not result in a loss of data.
Data Independence :- DBMS provides an abstract view of data that hides details of data representation and storage from end users. Data independence also means the capacity to change data kept at one place without changing data kept at other locations.
Providing Backup and Recovery :- A DBMS can provide facilities for recovering from hardware or software failures.

Derived Attribute: A derived attribute is an attribute of a table in a database that can be calculated from other attributes. It does not require to be stored in the table and can be computed on the fly when required. For example, a customer table might have a "Total_spent" attribute that is calculated from the sum of all orders made by that customer.

Domain: The domain of an attribute in a database is the set of allowed values for that attribute. It defines the data type and range of values that can be stored in that attribute. For example, if an attribute "Age" has a domain of INTEGER, it means that only integer values are allowed in that attribute.

Strong Entity: A strong entity is an entity in a database that has a primary key and can exist independently of other entities. A strong entity has its own unique identifier and can be identified and distinguished from other entities based on its primary key.

Superkey: A superkey is a set of attributes in a table that can be used to uniquely identify a record in the table. A superkey can be a single attribute or a combination of attributes. A primary key is a superkey that has been chosen by the database designer to be the unique identifier for the table.

Tuple: A tuple is a single row of data in a relational database table. It is a collection of values that represent a single record in the table. Each value in the tuple corresponds to an attribute of the table. For example, if a table has the attributes "ID", "Name", and "Address", a tuple might contain the values "1", "John Smith", and "123 Main St".

Data models define how the logical structure of a database is modeled. Data models define how data is connected to each other and how they are processed and stored inside the system.

1.The Main data model used in many current commercial DBMSs is the relational data model. It represents a database as a collection of tables, where each table can be stored as a separate file. Most relational databases use the high-level query language called SQL. Examples - MySQL, Oracle

2. The Object data model has been implemented in some commercial systems but has not had widespread use. The object data model defines a database in terms of objects, their properties, and their operations.

Two other important data models, now are the Network and Hierarchical models.
4. The Network model represents data as record types and represents a limited type of 1:N relationship. A 1:N, or one-to-many, relationship relates one instance of a record to many record instances using some pointer linking mechanism in these models. Examples - IDMS and SUPRA

Specialization is the process of defining a set of subclasses of an entity type which is called the superclass of the specialization. For example, the set of subclasses {SECRETARY, ENGINEER, TECHNICIAN} is a specialization of the superclass EMPLOYEE that distinguishes among employee entities based on the job type of each employee entity. Another specialization of the EMPLOYEE entity type may yield the set of subclasses {SALARIED_EMPLOYEE, HOURLY_EMPLOYEE}; this specialization distinguishes among employees based on the method of pay.

Generalization
The reverse process of abstraction in which we suppress the differences among several entity types, identify their common features, and generalize them into a single superclass of which the original entity types are special subclasses is called Generalization. For example, consider the entity types CAR and TRUCK . Because they have several common attributes, they can be generalized into the entity type VEHICLE.

One important task of a database administrator is to regularly backup the database. Then if the database ever becomes corrupted, the database administrator can use the backup files to restore the database.

It is important to make regular backups of all data in case of loss. Database recovery is the process of restoring the database to original state as it was before database failure occurs. mysqldump is an effective tool to backup MySQL database. It creates a *.sql file. # mysqldump -u root -p[root_password] [database_name] > dumpfilename.sql

When you are restoring the dumpfilename.sql on a remote database, make sure to create the database before you can perform the restore. # mysql -u root -p[root_password] [database_name] < dumpfilename.sql

In MySQL, a subquery is a query within a query. You can create subqueries within your SQL statements. A sub query is a select query that is contained inside another query. The inner select query is usually used to determine the results of the outer select query.

Types of Subqueries

1) A scalar subquery is a subquery that returns exactly one column value from one row.

2) An independent subquery is the subquery that return at most one value. SQL then compares the value that results from the subquery with the value on the other side of the comparison operator.

SELECT employee_id,name,salary FROM employees WHERE salary > (SELECT AVG(SALARY) FROM employees); 3) Multiple row subquery is the one that returns more than one row

4) A correlated subquery is a subquery that contains a reference to a table (in the parent query) that also appears in the inner query. A correlated subquery is evaluated once for each row in outer query

1. In SQL, a view is a virtual table based on the result-set of an SQL statement. By virtual, we mean, the tables do not store any data of their own but display data stored in other tables referenced in the SELECT statement. Views improve security of the database by showing only intended data to authorized users. They hide sensitive data.

2. Views make life easy as you do not have write complex queries time and again. It's possible to use INSERT, UPDATE and DELETE on a VIEW. These operations will change the underlying tables of the VIEW.

3. A view contains rows and columns, just like a real table. The fields in a view are fields from one or more real tables in the database.

4. CREATE VIEW view_name AS SELECT column1, column2, ... FROM table_name WHERE condition; A view can be updated with the CREATE OR REPLACE VIEW command. CREATE OR REPLACE VIEW view_name AS SELECT column1, column2, ... FROM table_name WHERE condition; A view is deleted with the DROP VIEW command. DROP VIEW view_name;

Database Administrators (DBAs) - These users have full control over the database and are responsible for managing and maintaining the database, including creating and modifying tables, managing user accounts, and performing backups and restores.

Application Users - These users are responsible for interacting with the database through an application, such as a website or software program. They typically have limited access to the database, with permissions to perform specific tasks such as querying, inserting, and updating data.

End/Naive Users - These users are the final users who access the database through an application or website. They typically have the least amount of access to the database and are only able to view or retrieve data.

Inner Join: An inner join in MySQL is used to retrieve only the matching records from two or more tables. The result of an inner join will include only the rows from both tables where the join condition is met. The syntax for an inner join in MySQL is as follows:

SELECT * FROM table1 INNER JOIN table2 ON table1.column = table2.column;

For example, imagine we have two tables: "customers" and "orders". The customers table has a column named "customer_id", and the orders table has a column named "customer_id" as well. We want to retrieve all the orders made by a specific customer.

SELECT * FROM orders INNER JOIN customers ON orders.customer_id = customers.customer_id WHERE customers.customer_id = 1;


Outer Join: An outer join in MySQL is used to retrieve all records from one table and the matching records from another table. If there is no matching record in the second table, the result will include a NULL value in the columns of the second table. The syntax for an outer join in MySQL is as follows:

SELECT * FROM table1 LEFT JOIN table2 ON table1.column = table2.column;

For example, imagine we have two tables: "employees" and "departments". The employees table has a column named "department_id", and the departments table has a column named "department_id" as well. We want to retrieve all the employees and their departments.

SELECT * FROM employees LEFT JOIN departments ON employees.department_id = departments.department_id;

This query will retrieve all the employees, even if they are not assigned to a department, in this case the department_id column will be null on the department side.
